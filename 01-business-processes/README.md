# BPMN-диаграмма процесса TO-BE: Внедрение ОТА-системы

## Описание артефакта
BPMN-диаграмма в нотации **BPMN 2.0**, описывающая целевой процесс **(TO-BE)** обновления бортового ПО электромобиля "Атом" после внедрения OTA-системы.

## Контекст
Проект выполнен для АО "Кама" в рамках дипломной работы в РЭУ им. Г.В. Плеханова. 
На момент анализа процесс обновления прошивок выполнялся вручную — через посещение дилерского центра, что занимало до 4 часов и требовало логистических затрат.

**Задача:** спроектировать процесс обновления "по воздуху" (Over-The-Air), исключив участие дилера.

## Что отражено на диаграмме
- Участники процесса (роли): владелец авто, OTA-облако, бортовой компьютер автомобиля
- Основные шаги: проверка версии → загрузка прошивки → валидация → установка → подтверждение
- Точки принятия решений (data-based gateway)
- Обработка ошибок и откат процесса при сбое

## Ценность для бизнеса
- Время обновления снижено с 4 часов до 5 минут
- Исключены логистические затраты на поездки в дилерский центр
- Процесс стал автоматизированным и отслеживаемым в реальном времени
